# Contributing to the Sprawl Demo DNA

First off, thanks for taking the time to contribute! 🎉

This repository is designed to be a safe, welcoming, and frictionless playground for newcomers to explore the Sprawl CLI. When contributing, keep the **DevRel 3-Minute Rule** in mind: any changes should maintain the zero-friction experience and allow users to go from discovery to running code in under 3 minutes.

## How Can I Contribute?

### Adding New Demo Assets
If you have an idea for a fun, harmless, and educational rule, persona, or workflow that demonstrates the power of the Atomic Agentic Fabric without requiring complex dependencies, we'd love to see it!

1. Fork the repository.
2. Create a new branch (`git checkout -b feature/cool-new-demo-skill`).
3. Add your asset to the appropriate folder (`atoms/`, `molecules/`, `skills/`, `rules/`, or `workflows/`).
4. Ensure your asset is fully documented and written in a welcoming, accessible tone.
5. Open a Pull Request!

### Improving Documentation
Found a typo? Have a better analogy to explain "Molecules"? Documentation improvements are highly appreciated. We want this to be the friendliest AI repository on GitHub.

## Styleguides
- **Tone:** Keep it crisp, objective, but deeply accessible. Avoid hard academic fluff or overly dense tech jargon.
- **Formatting:** Use standard Markdown. Make use of emojis where they add clarity or friendliness.
